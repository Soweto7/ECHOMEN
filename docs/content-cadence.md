# Founder-Led Content Cadence

## Weekly publishing rhythm
- **Mon:** Build log (what shipped, what broke, what changed)
- **Wed:** Failure -> fix thread (one concrete bug and resolution)
- **Fri:** User spotlight (workflow win, ROI, quote)

## Content formats
- X/Twitter thread
- LinkedIn post
- 60-90s product clip
- GitHub changelog linkback

## Source material pipeline
1. Pull highlights from `CHANGELOG.md`.
2. Extract one run report via **Share Run**.
3. Turn logs into before/after storytelling.
4. End each post with public roadmap item CTA.
